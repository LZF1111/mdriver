# MDriver 训练模式 / 自训练说明

> 一句话：模型权重**不变**，变的只有一包**外置领域知识（技能包）**。
> "训练" = 用客观信号 + 固定基准集，不断决定这包知识里**该加什么、删什么、谁更重要**。

---

## 一、两种学习方式（并存）

| | 被动学习 | 主动自训练（本次新增） |
|---|---|---|
| 触发 | 你跑真实任务时顺手沉淀 | 点「▶ 开始自训练」对固定基准集刷 |
| 数据来源 | 你的真实任务信号 | 你预设的基准任务 + 判据 |
| 目标 | 顺手积累 | **明确优化"通过率"** |
| 取舍 | 强信号自动沉淀 / 弱信号产候选待确认 | **keep-or-revert：没提升就整批回滚** |

两者写进**同一个**技能包，互相增强。

---

## 二、被优化的到底是什么

### 唯一标量目标 = 基准集通过率

```
通过率 = 通过的基准数 / 基准总数 × 100%
```

记录在 `skill.json` 的 `trainHistory[].passRate`，对应面板上蓝色曲线。**自训练的全部意义就是把这条曲线往上推。**

### 实际被调整的"旋钮"（技能包里的可变状态）

| 旋钮 | 字段 | 作用 |
|---|---|---|
| ① 经验条目（增/删） | `lessons[].text/fix` | 注入系统提示的"踩坑笔记" |
| ② 经验权重 | `lessons[].hits` | **命中越多排越前**；只注入 hits 前 15 条 |
| ③ 硬规则 | `lintRules[]` | lmp_lint 强制拦截（权重拉满的约束） |
| ④ 参数偏好 | `params{}` | 领域默认参数 |
| ⑤ 模板白名单 | `templates[]` | 已验证可复用脚本 |

> 核心：真正被训练动的是 **`lessons` 的内容 + `hits` 排序**。
> `benchmarks`（考卷）、`trainHistory`/`history`（曲线）、`stats`（统计）**只是测量/记录，不是被调对象**。

## 三、自训练闭环（带 keep-or-revert）

```
① 基线测评：用当前技能包跑一遍基准集，不加任何经验 → 得 baseline 通过率
② 每一轮：
   a. 快照当前 lessons
   b. 逐个跑基准；失败 → 提炼"具体坑→修法"经验写入（下一题即生效）
   c. 跑完算本轮通过率
   d. 通过率 > baseline → 保留，刷新 baseline
      通过率 ≤ baseline → 整批回滚（restoreLessons 还原到快照）
③ 记录"决策后保留的真实能力"到曲线
```

**意义**：每条新经验都必须用通过率证明自己有用，**没用的当场回滚**——"差的经验"被通过率自动淘汰，不用人管。

### "具体坑→修法"怎么提炼（`lessonsFromSignal`）

不是 AI 自由发挥，是**规则映射**：

1. **探针坏 verdict** → 查固定表：
   - `lost-atoms` → "建几何后先 minimize、检查初始重叠、非周期边界配 fix wall"
   - `temp-nan` → "timestep 太大或初速没设，先减小 timestep + velocity create"
   - `energy-spike` → "初始构型重叠，先 minimize 或拉大原子间距"
   - `timeout` → "体系偏大，先缩小规模或减步数验证物理再放大"
2. **lint 报过的规则** → 直接固化（带规则自带的 msg + fix）
3. **判据失败兜底** → 按判据类型给可执行建议（数值未达标 / 期望输出缺失 / 出现禁止信号 …）

最后按 `rule`/规范化文本**去重**。

## 四、基准判据类型（全部机器可判定）

| 类型 | 含义 |
|---|---|
| `probe_healthy` | 探针 verdict 必须 healthy |
| `lint_clean` | lmp_lint 必须无 error |
| `run_completed` | 异步完整 run 跑完（慢，依赖异步返回） |
| `output_present` | 工具输出须匹配正则 |
| `output_absent` | 工具输出须**不**出现正则（如 `ERROR\|lost atoms`） |
| `value_range` | 用正则捕获组 1 取数值，须 ∈ [min, max]（如 `R2=([0-9.]+)`） |

**通过 = 所有勾选/填写的判据全部满足。**

## 五、怎么用

1. 右下角「🧠 训练」→ 选/新建领域 → 启用。
2. 「🎯 自训练基准」→「➕ 添加基准任务」：
   - 填任务描述（会作为一次完整任务发给 agent）；
   - 勾选/填写"什么算通过"。
3. 设轮数 + 勾「留存检验」（默认开）→「▶ 开始自训练」。
4. 看蓝色**通过率曲线**上升；经验在「📚 已学经验」里查看（也可手动加/删/升级为硬规则）。

经验明文存盘：`skills/<领域名>/skill.json`，可直接编辑器打开看。

## 六、诚实的边界

- **不是梯度强化学习**：是 verbal / in-context RL（反思式），改的是外置知识不是模型权重。
- **keep-or-revert 是"批次级"信用分配**：一轮里加的多条经验整批保留或整批回滚，不做单条 A/B（那样要成倍的 LLM 调用，太贵）。
- **成本**：N 轮训练 ≈ (N+1) × 基准数 次完整任务（多一次基线）。基准别一次设太多。
- **`run_completed` 判据慢**：依赖异步任务在一轮内返回，快循环建议优先用 `probe_healthy`/`lint_clean`/输出正则/数值范围这类同步判据。
